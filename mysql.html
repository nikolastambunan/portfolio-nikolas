Memahami MySQL: Panduan Lengkap untuk Pemula

📅 10 Januari 2024 ⏱️ 15 menit baca 📖 Level: Pemula - Menengah

MySQL adalah sistem manajemen basis data relasional (RDBMS) sumber terbuka yang paling banyak digunakan di dunia. Dari aplikasi web sederhana hingga sistem enterprise kelas dunia, MySQL menjadi tulang punggung penyimpanan data. Artikel ini akan membawa Anda dari nol hingga mampu membuat dan mengelola database sendiri.

📌 Apa itu Database?

Database adalah kumpulan data terstruktur. Bayangkan seperti lemari arsip digital yang menyimpan informasi penting secara teratur, cepat diakses, dan mudah dikelola.

1. Mengapa MySQL Populer?

  • Gratis & Open Source — Tanpa biaya lisensi, komunitas besar.
  • Performa tinggi & handal — Cocok untuk skala kecil hingga besar.
  • Mudah dipelajari — Menggunakan SQL standar.
  • Dukungan luas — Dokumentasi lengkap dan banyak tutorial.

3. SQL Dasar: Manipulasi Data

Berikut empat operasi utama (CRUD) yang wajib dikuasai.

Membuat Database & Tabel

-- Buat database baru
CREATE DATABASE universitas;

-- Gunakan database
USE universitas;

-- Buat tabel mahasiswa
CREATE TABLE mahasiswa (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nim VARCHAR(20) UNIQUE NOT NULL,
    nama_lengkap VARCHAR(100) NOT NULL,
    jurusan VARCHAR(50),
    ipk DECIMAL(3,2) DEFAULT 0.00,
    tanggal_daftar DATE
);
💡 Tips Penting

Gunakan AUTO_INCREMENT untuk kolom ID agar otomatis bertambah. Tambahkan UNIQUE untuk memastikan tidak ada data duplikat (misalnya NIM).

Memasukkan Data (INSERT)

-- Tambah satu mahasiswa
INSERT INTO mahasiswa (nim, nama_lengkap, jurusan, ipk, tanggal_daftar)
VALUES ('2024001', 'Andi Saputra', 'Informatika', 3.75, '2024-08-01');

-- Tambah banyak sekaligus
INSERT INTO mahasiswa (nim, nama_lengkap, jurusan, ipk, tanggal_daftar) VALUES
('2024002', 'Budi Wijaya', 'Sistem Informasi', 3.50, '2024-08-02'),
('2024003', 'Citra Kirana', 'Informatika', 3.90, '2024-08-01'),
('2024004', 'Dian Permatasari', 'Manajemen', 3.60, '2024-08-03');

Membaca Data (SELECT) — Contoh Beragam

-- Semua data
SELECT * FROM mahasiswa;

-- Kolom tertentu dengan alias
SELECT nim AS NIM, nama_lengkap AS Nama, ipk FROM mahasiswa;

-- Filter dengan WHERE
SELECT * FROM mahasiswa WHERE jurusan = 'Informatika' AND ipk > 3.6;

-- Pencarian teks
SELECT * FROM mahasiswa WHERE nama_lengkap LIKE '%andi%';

-- Sorting & limit
SELECT * FROM mahasiswa ORDER BY ipk DESC LIMIT 3;

Memperbarui Data (UPDATE)

-- Update ipk mahasiswa dengan nim tertentu
UPDATE mahasiswa 
SET ipk = 3.80, jurusan = 'Teknik Komputer'
WHERE nim = '2024001';

-- Update banyak baris (hati2 dengan WHERE)
UPDATE mahasiswa SET ipk = ipk + 0.05 WHERE jurusan = 'Informatika';
⚠️ Peringatan!

Jika tidak menggunakan WHERE pada UPDATE atau DELETE, semua baris dalam tabel akan terpengaruh. Selalu cek kondisi sebelum mengeksekusi.

Menghapus Data (DELETE)

-- Hapus satu mahasiswa
DELETE FROM mahasiswa WHERE id = 4;

-- Hapus berdasarkan kondisi
DELETE FROM mahasiswa WHERE ipk < 2.0;

4. Relasi Antar Tabel (Contoh Praktis)

Hubungan antar tabel membuat database lebih powerful. Misalnya di universitas:

One-to-Many: Satu Dosen Mengajar Banyak Mata Kuliah

CREATE TABLE dosen (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nama_dosen VARCHAR(100) NOT NULL,
    nip VARCHAR(20) UNIQUE
);

CREATE TABLE mata_kuliah (
    id INT PRIMARY KEY AUTO_INCREMENT,
    kode_mk VARCHAR(10) UNIQUE NOT NULL,
    nama_mk VARCHAR(100) NOT NULL,
    sks INT,
    dosen_id INT,
    FOREIGN KEY (dosen_id) REFERENCES dosen(id) ON DELETE SET NULL
);

Many-to-Many: Mahasiswa Mengambil Banyak Mata Kuliah

-- Tabel penghubung (enrollment)
CREATE TABLE krs (
    mahasiswa_id INT,
    mata_kuliah_id INT,
    semester VARCHAR(10),
    nilai_akhir CHAR(2),
    PRIMARY KEY (mahasiswa_id, mata_kuliah_id),
    FOREIGN KEY (mahasiswa_id) REFERENCES mahasiswa(id),
    FOREIGN KEY (mata_kuliah_id) REFERENCES mata_kuliah(id)
);

5. Query yang Sering Dipakai (Agregasi & JOIN)

Data akan lebih bermakna dengan agregasi dan penggabungan tabel.

Menghitung & Mengelompokkan

-- Jumlah mahasiswa per jurusan
SELECT jurusan, COUNT(*) AS jumlah_mahasiswa
FROM mahasiswa
GROUP BY jurusan;

-- Rata-rata IPK per jurusan (hanya jurusan dengan ipk > 3.4)
SELECT jurusan, AVG(ipk) AS rata_ipk
FROM mahasiswa
GROUP BY jurusan
HAVING rata_ipk > 3.4;

-- Nilai maksimum dan minimum
SELECT MAX(ipk) AS ipk_tertinggi, MIN(ipk) AS ipk_terendah FROM mahasiswa;

JOIN Tabel (Contoh Relasi)

-- Menampilkan KRS lengkap: nama mahasiswa, mata kuliah, dan nilai
SELECT m.nama_lengkap, mk.nama_mk, k.nilai_akhir
FROM mahasiswa m
JOIN krs k ON m.id = k.mahasiswa_id
JOIN mata_kuliah mk ON k.mata_kuliah_id = mk.id
WHERE k.semester = 'GANJIL 2024';

-- LEFT JOIN: semua dosen, meski tidak mengajar matkul
SELECT d.nama_dosen, mk.nama_mk
FROM dosen d
LEFT JOIN mata_kuliah mk ON d.id = mk.dosen_id;

6. Studi Kasus: Sistem Peminjaman Buku Perpustakaan

Mari terapkan dalam skenario nyata.

CREATE DATABASE perpustakaan;
USE perpustakaan;

-- Buku
CREATE TABLE buku (
    id INT PRIMARY KEY AUTO_INCREMENT,
    isbn VARCHAR(13) UNIQUE,
    judul VARCHAR(200) NOT NULL,
    penulis VARCHAR(100),
    stok INT DEFAULT 1
);

-- Anggota
CREATE TABLE anggota (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nama_anggota VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    tgl_daftar DATE DEFAULT (CURRENT_DATE)
);

-- Peminjaman
CREATE TABLE peminjaman (
    id INT PRIMARY KEY AUTO_INCREMENT,
    anggota_id INT,
    buku_id INT,
    tgl_pinjam DATE NOT NULL,
    tgl_kembali DATE,
    FOREIGN KEY (anggota_id) REFERENCES anggota(id),
    FOREIGN KEY (buku_id) REFERENCES buku(id)
);

-- Insert data contoh
INSERT INTO buku (isbn, judul, penulis, stok) VALUES
('9786020331957', 'Atomic Habits', 'James Clear', 3),
('9786020331964', 'Deep Work', 'Cal Newport', 2);

INSERT INTO anggota (nama_anggota, email) VALUES
('Rina Oktavia', 'rina@email.com'),
('Eko Prasetyo', 'eko@email.com');

-- Peminjaman buku
INSERT INTO peminjaman (anggota_id, buku_id, tgl_pinjam) VALUES
(1, 1, '2024-01-10'),
(2, 2, '2024-01-12');

-- Query laporan peminjaman aktif
SELECT a.nama_anggota, b.judul, p.tgl_pinjam
FROM peminjaman p
JOIN anggota a ON p.anggota_id = a.id
JOIN buku b ON p.buku_id = b.id
WHERE p.tgl_kembali IS NULL;

7. Praktik Terbaik (Best Practices)

  • Gunakan nama tabel dan kolom yang jelas — hindari singkatan membingungkan.
  • Selalu tentukan PRIMARY KEY untuk setiap tabel.
  • Indeks pada kolom yang sering dicari — contoh: email, NIM.
  • Backup rutin dengan mysqldump untuk mencegah kehilangan data.
  • Gunakan prepared statements pada aplikasi untuk menghindari SQL injection.
  • Jangan sembarangan menghapus data, lebih baik tambahkan kolom is_deleted (soft delete).
